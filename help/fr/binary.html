Si vous ne vous sentez pas à l'aise pour compiler XFCE vous-même, une distribution déjà compilée est disponible.

          1) Télécharger la distribution
           

          La distribution précompilée de XFCE est un fichier tar compréssé avec gzip. L'application est compilée avec la librairie XForms en statique, ce qui fait qu'il n'est même pas nécessaire d'avoir XForm installé sur le système pour pouvoir utiliser cette distribution.

          XFce est disponible pour :

          • Linux libc5
          • Linux glibc (alias libc6)
          • Solaris 2.6 Intel Edition
          • FreeBSD


          2) Installer la distribution

          La distribution est maintenant en deux parties, pour economiser de la place disque :

          • Les données communes (sons, fonds d'écran, icônes)
          • Les fichiers binaires spécifiques a la plateforme


          L'archive contient déjà les chemins d'accès corrects, il suffit donc de la décomprésser depuis la racine de l'arborescence du système :
           

            $ gunzip xfce-common-2.?.?.bin.tar.gz
            $ tar xvf xfce-common-2.?.?.bin.tar -C /
            $ gunzip xfce-2.?.?.<platform>.static.bin.tar.gz
            $ tar xvf xfce-2.?.?.<platform>.static.bin.tar -C /
             
           3) Pour les utilisateurs de RPM (le format d'archives de Red Hat)

          Quelques rappels des commandes de bases de rpm (pour plus d'information sur RPM, utilisez man rpm ou rpm --help) :
           

          • rpm -ivh <archive>.i386.rpm

            • Installe l'archive
          • rpm -U <archive>.i386.rpm

            • Mise a jour d'une archive
          • rpm -e <archive>

            • Supprime l'archive et ses fichiers
          • rpm -ba <spec>

            • Reconstruit entièrement l'archive à partir d'un fichier spec
             
          La distribution rpm de XFce pour Linux Red Hat est aussi en deux parties. En conséquence, vous devrez installer les données communes avant les fichiers binaires.
